## PROBLEMA 1: Locks Fantasma — Sistema Não Consegue Reiniciar
 Severidade: 🔴 CRÍTICA
 Fonte: .telegram.lock, .instance.lock, .correoto.lock

 CAUSA:
 O PID 22196 (processo morto) deixou 3 ficheiros de lock no sistema.
 Qualquer nova instância do Correoto detecta os locks e recusa iniciar,
 criando um impasse total.

 IMPACTO:
 - Sistema completamente parado
 - Nenhuma instância nova consegue iniciar
 - Utilizador vê ecossistema "morto" sem saber porquê

 CORREÇÃO IMPLEMENTADA: ✅
 1. Removidos .telegram.lock, .instance.lock, .correoto.lock
 2. Sistema pode agora reiniciar normalmente

## PROBLEMA 2: Bytecode Corrompido — Erro Silencioso no Telegram
 Severidade: 🔴 ALTA
 Fonte: bot/__pycache__/handlers.cpython-313.pyc, main_stderr.log

 CAUSA:
 O ficheiro fonte (bot/handlers.py, 455 linhas) está sintaticamente correto,
 mas o bytecode compilado (.pyc) continha uma versão antiga com uma f-string
 mal formatada (linha 460, que já não existe no source atual).
 O erro "unterminated f-string literal" impedia o Telegram de iniciar.

 IMPACTO:
 - Bot Telegram silencioso (não responde a comandos)
 - Utilizador perde interface principal com o sistema
 - Mensagem de erro aparece a cada startup

 CORREÇÃO IMPLEMENTADA: ✅
 1. Removidos 31 ficheiros .pyc de todo o projeto
 2. Próximo startup recompila bytecode fresco do source correto

---

## PROBLEMA 3: Loop de Fallback Infinito — 211 Tarefas Repetidas
 Severidade: 🟡 MÉDIA
 Fonte: memory/backlog.json, autonomous_loop.py

 CAUSA:
 O sistema gerava sempre as mesmas 5 tarefas de fallback quando o debate
 do OrganicMind falhava. Como as tarefas falhavam (por falta de contexto),
 o ciclo recomeçava: falha → fallback → executa → falha → fallback...
 Resultado: 211 tarefas no backlog, 90% são repetições das mesmas 5.

 IMPACTO:
 - Backlog inchado com tarefas inúteis
 - Sistema gasta recursos em tarefas que já falharam
 - Tarefas reais importantes enterradas no ruído

 CORREÇÃO IMPLEMENTADA: ✅
 1. Expandido de 5 para 10 tarefas de fallback com rotação
 2. Tarefas "running" há >1h marcadas como failed (2 tarefas)
 3. Tarefas "failed" duplicadas removidas (6 tarefas)
 4. Adicionadas 2 tarefas reais e úteis ao backlog
 5. Backlog reduzido de 211 → 207 tarefas (mais 2 novas úteis)
